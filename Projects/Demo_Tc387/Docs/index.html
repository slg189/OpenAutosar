项目概述

OpenAutosar 是一个面向量产的 AUTOSAR Classic Platform 嵌入式工程框架， 按照 AUTOSAR 分层模型组织代码：上层应用软件 (ASW)、基础软件 (BSW)、复杂驱动 (CDD)、 公共组件 (Components) 与微控制器抽象层 (MCAL)。每一层既支持 源码形式 （inc/ 头文件 + src/ 静态代码），也支持 预编译交付形式 （供应商 .a 库 + inc/ 头文件）。

AUTOSAR CP AURIX TC387 / TC367 SCons 构建 EB tresos / ETAS / Vector GoogleTest + QEMU MISRA C:2012

分层架构

自上而下的软件分层（CDD 可跨层直达硬件）：

ASW  应用软件 (VehicleControl / EnergyManagement …)
RTE  运行时环境 (Gen / 工具生成)
BSW  基础软件 (ETAS RTA-OS / Vector MICROSAR COM …)
CDD  复杂驱动 (CddPwm / CddMotor / CddSent)
MCAL  微控制器抽象 (Adc / Can / Port)
Hardware  Infineon AURIX TC3xx

Components 提供跨层公共代码（类型、编译器抽象、CRC 等）与启动模块 Sbm； Projects 负责把各层产物（.a）与生成的配置代码链接为最终镜像。

ASW · 应用层

存放上层各应用模型的代码。每个应用既可以 源码形式 组织 （模块文件夹下 src/ 源码 + inc/ 头文件 + SConscript）， 也可以 预编译形式 交付（模块根目录下 .a 库 + inc/ 头文件）。 ASW/inc/ 放置应用层公共对外头文件（如共享 RTE 类型）。

路径形式说明
ASW/inc/头文件公共对外类型 (Rte_Type.h)
ASW/VehicleControl/源码整车控制：inc/ + src/ + SConscript
ASW/EnergyManagement/预编译能量管理：inc/ + libEnergyManagement.a

BSW · 基础软件

项目 BSW 供应商源码。不再分 Vector/Etas 目录，直接按模块名（Os/Com/Crc/…）组织； 供应商、芯片、交付版本编码进 git 子库名（BSW_<Vendor>_<Chip>_<Delivery>，归 BSW group）。 每模块 src/ + inc/ + SConscript，编译产物 .a 释放到 BSW_Libs/（非源码可见者只拿二进制）；BSW/inc/ 为跨模块公共头文件。

BSW/Os/

ETAS RTA-OS：src/Os.c + inc/Os.h
子库 BSW_Etas_TC387_RtaOs → BSW_Libs/Os/libRtaOs.a

BSW/Com/

Vector MICROSAR COM：src/Com.c + inc/Com.h
子库 BSW_Vector_TC387_CBDxxxxxx → BSW_Libs/Com/libCom.a

BSW/inc/

公共对外接口 (EcuM.h 等)

CDD · 复杂驱动

各模块为一个复杂驱动 (Complex Device Driver)，可直接访问硬件外设。 源码模块在子目录下分 inc/（头文件）、src/（静态代码）、 gen/（配置代码），模块根目录提供 SConscript 构建脚本； 预编译模块则在模块根目录放 .a 库 + inc/ 头文件。

模块组成说明
CddPwm/inc/ src/ gen/ SConscriptGTM PWM 输出
CddMotor/inc/ src/ gen/ SConscript电机控制
CddSent/inc/ + libCddSent.aSENT 接收（预编译）

Components · 组件

组件模块代码与公共代码。其中 Mcu/Sbm/ 为启动代码模块 （Startup / Secure Boot Manager），在 main 之前完成时钟、看门狗、 内存初始化与应用镜像校验。同样支持 .a + inc/ 预编译形式。

路径说明
Components/Common/Std_Types / Compiler / Platform_Types 抽象，公共工具 (CRC)
Components/Mcu/Sbm/启动代码模块：inc/ + src/ + SConscript

MCAL · 微控制器抽象层

按芯片型号划分（Tc387/、Tc367/），每个驱动模块以 inc/ + src/ 源码形式，或 .a + inc/ 预编译形式提供。MCAL/inc/ 放公共标准类型。 本工程中 MCAL 的配置代码由 EB tresos 依据 Projects/<P>/Cfg/MCAL/*.arxml 生成至 Projects/<P>/Gen/MCAL/。

芯片模块形式
Tc387Adc / Can / Portinc/ + src/
Tc367Adcinc/ + src/
Tc367Caninc/ + libCan.a (预编译)

构建与测试流程

构建

  REM 1. 配置 TriCore 工具链环境
call Tools\Compiler\setup_env.bat

REM 2. 进入工程并构建 (SCons)
cd Projects/Demo_Tc387
scons -j %NUMBER_OF_PROCESSORS%
REM 产物: Out/Demo_Tc387.elf / .hex / .map

构建顺序：各模块 SConscript 编译为 .a → 安装到 Libs/ → 与 Gen/ 配置代码、供应商 BSW 库一同按 Ld/ 链接脚本链接为 ELF → objcopy 生成 HEX。

测试

  run_gtest.bat                       REM 单元测试 + OpenCppCoverage 覆盖率
run_integrationTest.bat Demo_Tc387  # QEMU 集成测试 (JUnit 报告)

约定与规范

  • 模块两种形态：源码形态（inc/ + src/[+ gen/] + SConscript） 或预编译形态（模块根 .a + inc/ 头文件）。
  • 配置与代码分离：工程配置 (Cfg/, arxml) → 工具生成 (Gen/) → 参与构建。
  • 产物隔离：Obj/、Out/、Gen/、Libs/*.a 不入库。
  • 代码质量：统一 .clang-format 风格，MISRA C:2012 + QAC 静态检查。
  • 命名：遵循 AUTOSAR 规范（如 <Module>_<Function>、Std_ReturnType）。
