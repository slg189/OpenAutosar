方案目标

满足两类需求：

  1. 信息安全分级：不同角色只能看到与其职责相关的源码，其余以二进制库（.a + 头文件）形式集成。
    • ASW 工程师：只看 ASW 模型代码；BSW、CDD 以库释放。
    • 基础（集成）工程师：只看 Projects/<P>/Integration/ 集成代码；ASW/BSW/CDD/MCAL 全部为 .a。
  2. 灵活编译：每个模块可独立配置 源码编译 / 库集成、是否启用、模块级编译选项，并支持命令行临时覆盖。

库化释放区 *_Libs

根目录新增四个释放区，存放各层对外交付的 inc/ 头文件与预编译 .a，不含源码。 由 release_libs.bat 从源码编译后回灌（也可由 CI 产出）。

OpenAutosar/ ├── ASW_Libs/ # ASW 释放(按项目, 子库 asw_libs_<P>): <Model>/inc + lib<Model>.a ├── BSW_Libs/ # BSW 释放(按项目, 子库 bsw_libs_<P>): Etas|Vector/ (inc + .a, 含配置代码) ├── CDD_Libs/ # CDD 释放(按项目, 子库 cdd_libs_<P>): <Cdd>/inc + lib<Cdd>.a ├── MCAL_Libs/ # MCAL 释放(按项目, 子库 mcal_libs_<P>): <Chip>/<Mod>/.. (.a 含配置+静态) │ ├── ASWs/ BSW/ CDDs/ MCAL/ # 源码 (按 manifest 选择性下发) └── Projects/Demo_Tc387/Integration/ # 集成代码 (基础工程师)

命名规则：MCAL 按 <Chip>/<Module>/ 镜像；CDD/ASW 按 <Module>/； BSW 按供应商 Etas/、Vector/。每个模块目录含 inc/ 与 lib<Name>.a。

Integration 集成层

在工程下新增 Projects/<P>/Integration/，存放基础工程师维护的集成 / 胶水代码： main.c、RTE 连接、EcuM 回调、任务体等。它通过各层 头文件 调用以 .a 释放的功能。

Projects/Demo_Tc387/Integration/ ├── inc/IntegrationCfg.h ├── src/main.c # 启动 + OS 起调 ├── src/Rte.c # 连接 ASW(.a) 与 BSW/CDD(.a) ├── src/EcuM_Callout.c # 初始化挂载点 └── SConscript # 编入工程, 链接 *_Libs/*.a

manifest 视图切换

用 google-repo manifest（manifests/）控制实际签出范围，从源头实现隔离： 不该看的层只发布 *_libs_*.git（仅 .a+头文件），不发布源码仓库。

Manifest角色源码仓库仓 (.a+inc)
default.xml平台/集成负责人全部—
manifest-asw.xmlASW 工程师ASWs, Components, Projects, ToolsBSW_Libs, CDD_Libs, MCAL_Libs
manifest-integration.xml基础工程师Projects, ToolsASW_Libs, BSW_Libs, CDD_Libs, MCAL_Libs
  REM 以 ASW 工程师视图签出
set MANIFEST_NAME=manifest-asw.xml && repo_metasar.bat init
repo_metasar.bat sync
隔离边界在 仓库下发 层面：拿不到源码仓库，自然看不到源码； *_Libs 作为受控发布物随对应库仓下发。

灵活编译 · module_config.py

每个模块独立声明：mode（source / lib）、enabled、cflags（模块级编译选项）。

  MODULES = {
  'MCAL': {
    'Adc':  {'mode': 'source', 'enabled': True, 'cflags': ['-O3']},
    'Can':  {'mode': 'lib',    'enabled': True},
    'Port': {'mode': 'source', 'enabled': True},
  },
  'CDD':  { 'CddPwm': {'mode':'lib'}, 'CddMotor': {'mode':'lib'} },
  'ASW':  { 'VehicleControl': {'mode':'source'},
            'EnergyManagement': {'mode':'lib'} },
  'BSW':  { 'Os': {'mode':'lib'}, 'Com': {'mode':'lib'} },
}
  • mode=source：从对应层源码编译为 .a，并可释放回 *_Libs/。
  • mode=lib：直接链接 *_Libs/ 下已释放的 .a（源码不可见时的默认形态）。
  • cflags 在全局选项之后追加，实现 按模块差异化编译（如对 ADC 单独 -O3）。

命令行灵活编译

命令作用
scons按 module_config.py 构建
scons variant=Debug切换 Debug / Release 全局选项
scons chip=Tc367切换目标芯片 (MCAL 源/库路径随之切换)
scons only=Adc,CddPwm仅编译指定模块 (白名单)
scons skip=EnergyManagement跳过指定模块 (黑名单)
scons layers=ASW,MCAL仅编译指定层
scons release编译并把源码模块释放回 *_Libs

构建与释放流程

module_config.py + CLI→ SConstruct 逐模块分流→ source: 编译 .a/lib: 取 *_Libs/.a→ + Integration objs + Gen→ 链接 ELF/HEX

释放（供下游二进制集成）：

  REM 把各层源码编译并连同头文件释放到根 *_Libs/
release_libs.bat all
set CHIP=Tc367 && release_libs.bat MCAL   # 仅释放 TC367 的 MCAL

常用示例

  REM ASW 工程师: 只联调本层模型, 其余用库
cd Projects/Demo_Tc387
scons layers=ASW variant=Debug

REM 基础工程师: 全部用库, 只编集成层 (其余 mode 已是 lib)
scons                          # Integration + *_Libs/*.a

REM 单独高优化编译某 MCAL 模块
scons only=Adc chip=Tc387      # Adc 带 module_config 中的 -O3

REM 发布: 编译 + 回灌库
scons release && release_libs.bat all
