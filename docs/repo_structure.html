划分原则

  • 源码 每个顶层文件夹一个子库（ASWs/、BSW/、CDDs/、MCAL/、Components/、Projects/、Tools/、Test/、docs/）。
  • 共享库 MCAL_Libs/（平台/芯片级）一个子库，跨项目共享。
  • 项目库 ASW_Libs/、CDD_Libs/、BSW_Libs/ 与项目强相关，按项目各建一个子库：asw_libs_<Project>、cdd_libs_<Project>、bsw_libs_<Project>（扁平命名、下划线分隔）。
  • 元信息 清单与根脚本/配置归 meta 子库（或 manifest 仓）。
为什么 ASW/CDD/BSW 按项目分库？它们的发布物随项目而异——ASW 模型、CDD 复杂驱动， 以及 BSW 的 .a（已把该项目的 ECUC 配置生成代码一并打包）。 按项目分库可做到：① 不同项目的库互不可见、独立授权；② 一个项目升级不影响他人；③ 配合信息安全分级，只给某项目的工程师下发其对应项目库。

项目身份只在 git 子库名 中（<layer>_libs_<project>）， 工作区路径直接是 <Layer>_Libs/<module>，不带项目子目录。 构建侧 build_helpers.lib_file 据此解析为 <Layer>_Libs/<name>/lib<name>.a； release_libs.bat ASW|CDD|BSW 释放到 <Layer>_Libs/ 根。

创建子库

用 Tools/Git/init_subrepos.bat 把当前 monorepo 各文件夹打包为独立子库并推送。 固定子库取自 repo_map.txt；项目库把整个 ASW_Libs/、CDD_Libs/、BSW_Libs/ 目录分别打包为 asw_libs_<PROJECT> / cdd_libs_<PROJECT> / bsw_libs_<PROJECT> （--project 指定，默认 Demo_Tc387）。

REM 预览将创建的子库 (不写任何东西)
Tools\Git\init_subrepos.bat https://github.com/<owner> --project Demo_Tc387 --dry-run

REM 本地初始化 + 推送 (远端空仓库需已存在)
Tools\Git\init_subrepos.bat https://github.com/<owner> --project Demo_Tc387 --push
脚本在 .subrepos\<name> 临时目录内 git init 并提交，避免在工作树里 产生嵌套 .git，不影响当前 monorepo。

在 GitHub 上建仓 + 推送

init_subrepos.bat 可在推送前自动创建 GitHub 仓库：优先 gh CLI，无 gh 时回退 REST API（需 GITHUB_TOKEN）。

REM 方式 A: gh CLI (推荐) —— 先 gh auth login
Tools\Git\init_subrepos.bat https://github.com/<user> --github <user> --push
REM 组织账户加 --org; 公开仓加 --public
Tools\Git\init_subrepos.bat https://github.com/<org> --github <org> --org --public --push

REM 方式 B: REST API (无 gh) —— 用 PAT
set GITHUB_TOKEN=ghp_xxxxxxxx
Tools\Git\init_subrepos.bat https://github.com/<owner> --github <owner> --push

手动建单个仓库：gh repo create <owner>/asw_libs_Demo_Tc387 --private， 或 curl -H "Authorization: token %GITHUB_TOKEN%" https://api.github.com/user/repos -d "{\"name\":\"asw_libs_Demo_Tc387\",\"private\":true}"。

新增一个项目时

  1. 用该项目的工作区（其 *_Libs/ 为该项目内容）；release_libs.bat ASW|CDD|BSW 回灌到 ASW_Libs/、CDD_Libs/、BSW_Libs/ 根。
  2. Tools/Git/init_subrepos.bat <base> --project <NewProj> --github <owner> --push 创建 asw_libs_<NewProj>、cdd_libs_<NewProj>、bsw_libs_<NewProj> 子库。
  3. 为该项目准备一套清单（或在 default 中把 *_libs_<NewProj> 映射到 <Layer>_Libs），切换项目即换这套 *_libs_<Proj> 子库。
